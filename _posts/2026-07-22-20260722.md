컴퓨팅 환경의 복잡성이 증가함에 따라, 현대 IT 인프라는 더 높은 수준의 유연성과 확장성을 요구받고 있다. Composable Infrastructure는 이러한 요구를 충족시키기 위해 등장한 개념으로, 하드웨어 및 소프트웨어 리소스를 동적으로 구성하여 필요에 따라 조합하고 재사용할 수 있는 방식을 제공한다. 이 문서에서는 WebAssembly(WASM)와 Kubernetes를 활용하여 Composable Infrastructure를 구현하고 오케스트레이션하는 방법을 다룬다.

<!--more-->

## 기술 개요 및 핵심 원리

### Composable Infrastructure란 무엇인가
Composable Infrastructure는 물리적 및 가상 리소스를 논리적으로 분리하고, 이를 소프트웨어 정의 방식으로 조합하여 특정 워크로드에 최적화된 환경을 제공하는 기술이다. 이를 통해 리소스의 활용도를 극대화하고, 운영 효율성을 높일 수 있다.

### WebAssembly(WASM)의 역할
WASM은 경량화된 바이너리 포맷으로, 브라우저뿐만 아니라 서버 및 엣지 환경에서도 실행 가능한 범용 실행 플랫폼이다. WASM은 높은 성능과 플랫폼 독립성을 제공하며, 다양한 언어로 작성된 코드를 실행할 수 있는 장점을 가진다. Composable Infrastructure에서는 WASM을 통해 경량화된 모듈을 생성하고, 이를 동적으로 조합하여 애플리케이션을 구성할 수 있다.

### Kubernetes와의 통합
Kubernetes는 컨테이너화된 애플리케이션의 배포, 확장 및 관리를 자동화하는 오픈소스 플랫폼이다. WASM 모듈을 Kubernetes 클러스터 내에서 관리하면, 기존 컨테이너 기반 워크로드와 WASM 기반 워크로드를 통합하여 운영할 수 있다. 이를 통해 Composable Infrastructure의 유연성과 확장성을 극대화할 수 있다.

## 실무에서 검증된 코드 구현체와 사용 가이드

### WASM 모듈 생성 및 배포
다음은 Rust로 작성된 WASM 모듈을 생성하고 Kubernetes에서 실행하는 과정을 다룬다.

#### 1. Rust 환경 설정 및 WASM 모듈 생성
Rust를 사용하여 간단한 WASM 모듈을 생성한다.

bash
# Rust 설치
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh

# WASM 타겟 추가
rustup target add wasm32-unknown-unknown
아래는 "Hello, World!"를 출력하는 WASM 모듈의 코드이다.

rust
// src/lib.rs
#![no_std]

#[no_mangle]
pub extern "C" fn hello_world() -> *const u8 {
    b"Hello, World!\0".as_ptr()
}
컴파일 명령:

bash
cargo build --target wasm32-unknown-unknown --release
#### 2. WASM 모듈 컨테이너화
Docker 이미지를 생성하여 WASM 모듈을 Kubernetes에서 실행 가능하도록 준비한다.

Dockerfile
# Dockerfile
FROM scratch
COPY target/wasm32-unknown-unknown/release/hello_world.wasm /app/hello_world.wasm
CMD ["wasmtime", "/app/hello_world.wasm"]
이미지 빌드 및 푸시:

bash
docker build -t <your-dockerhub-username>/hello-wasm:latest .
docker push <your-dockerhub-username>/hello-wasm:latest
#### 3. Kubernetes에 배포
Kubernetes Deployment YAML 파일을 작성한다.

yaml
# deployment.yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: hello-wasm
spec:
  replicas: 1
  selector:
    matchLabels:
      app: hello-wasm
  template:
    metadata:
      labels:
        app: hello-wasm
    spec:
      containers:
      - name: hello-wasm
        image: <your-dockerhub-username>/hello-wasm:latest
        ports:
        - containerPort: 8080
배포 실행:

bash
kubectl apply -f deployment.yaml

## 적용 시 장단점 및 고려사항

### 장점
1. **유연성**: WASM 모듈은 경량화되어 있어 빠르게 로드 및 실행 가능하며, 다양한 환경에서 동일하게 동작한다.
2. **보안성**: WASM은 샌드박스 환경에서 실행되므로, 실행 중인 코드를 격리하여 보안성을 높인다.
3. **효율성**: WASM의 경량성과 성능은 기존 VM이나 컨테이너 대비 리소스 소모를 줄이고, 빠른 실행 속도를 제공한다.

### 단점
1. **도구 및 생태계의 미성숙**: WASM과 Kubernetes의 통합은 아직 초기 단계로, 관련 도구와 생태계가 완전하지 않다.
2. **디버깅 어려움**: WASM 모듈의 디버깅은 기존 컨테이너 기반 워크로드에 비해 복잡하다.

### 고려사항
1. WASM 모듈의 크기를 최소화하고, 필요한 의존성을 명확히 정의해야 한다.
2. Kubernetes 클러스터에서 WASM 워크로드를 실행하려면, WASM 런타임(예: Wasmtime, Wasmer)을 컨테이너에 포함시켜야 한다.
3. 기존 컨테이너 기반 애플리케이션과 WASM 기반 애플리케이션의 통합 전략을 사전에 설계해야 한다.

---

Composable Infrastructure는 WASM과 Kubernetes의 결합을 통해 새로운 차원의 유연성과 확장성을 제공한다. 그러나 기술적 한계와 생태계의 미성숙함을 고려하여, 특정 워크로드에 적합한 방식으로 점진적으로 도입하는 것이 바람직하다.
